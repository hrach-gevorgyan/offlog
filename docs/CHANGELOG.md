# Offlog — Changelog

The single source of truth for version history — this used to be duplicated
across `TECH.md` (technical detail) and the root `README.md` (short
highlights), which had drifted in coverage (TECH.md's table was more
complete). Consolidated here 2026-07 as part of the documentation cleanup;
both `README.md` and `docs/TECH.md` now link here instead of keeping their
own copy. New entries go at the top, one row per released version.

This table keeps the newest ~8 releases in full detail. Older releases are
compressed to one line each in [CHANGELOG-ARCHIVE.md](CHANGELOG-ARCHIVE.md)
(git still has every full diff) — split out 2026-07 once this file's
default-loaded size grew large enough to matter. When adding a new entry,
move the oldest row here into the archive (compressed to one line) so this
table stays roughly the same length release over release.

| Version | Changes |
|---|---|
| **4.10.0** | **Dashboard weekly stats + command palette** (B17, B9). **B17**: `getDashboardData()` now also returns `completedLast7Days`, `busiestProjectName`, and `todayTasks` — a "N completed this past week · busiest: X" line under the Dashboard header, plus a new "Today" panel alongside the existing Pinned/Overdue ones. "Completed" is derived from the existing positional last-column check plus `updated_at` within 7 days rather than log docs, since move-action log entries only store the target column's *name* (fragile against renames), not its id — noted tradeoff: `updated_at` bumps on any edit, so a task completed earlier but merely edited within the window could false-positive, acceptable for a glance-level stat. **B9**: Ctrl+K's existing `GlobalSearch` overlay now also matches action commands (go to Dashboard/Focus/Agenda, Quick Add, toggle theme, toggle high contrast, open Settings/Changelog/Deleted, Sync Now) via a small fixed list in new `commands.ts`, matched by the same plain substring logic `searchAllTasks()` already uses — no fuzzy library added. Commands and task results share one combined keyboard-navigable list, commands shown first; Settings/Changelog/Deleted route through a `bind:this` ref into `Sidebar.svelte` rather than lifting that modal state up a level |
| **4.9.0** | **Archive-adjacent housekeeping** (B27, B32, B15). **B27**: archived tasks were previously only reachable via a toggle buried inside List view — Dashboard's header now shows a DB-wide archived-task count, and List view's own "Archived" toggle carries a live per-project count badge even before it's opened. **B32**: projects can now be archived as a whole (`ProjectDoc.archived`, `archiveProject()`/`unarchiveProject()`/`getArchivedProjects()`) — archives the project doc plus its non-done tasks, soft only, restorable. Deliberately **not** a sidebar icon (the sidebar was already called out as visually overloaded once, v4.6.5) or an inline picker bolted into the Organize pane (rejected mid-build, felt inconsistent with the rest of Settings) — landed as a new **Archived Projects** manager panel (`ArchivedProjectsManager.svelte`), opened the same way as Manage Spaces/Tags/Custom Fields, with an archive picker, a restore list, and a direct per-project Delete action (owner-requested) so a project doesn't need restoring first just to delete it. A real cross-browser bug was caught along the way: native `<select>` popups render on an OS-controlled light background regardless of the app's dark theme, and the select's inherited `--text` color (pale, for dark mode) was rendering as near-invisible light-on-white inside the open dropdown — fixed with `color-scheme: light` plus an explicit dark `option` color. **B15**: `MaintenanceModal.svelte`'s step list/progress bar/Run button now render directly inside Settings → Maintenance's own detail pane instead of a second modal-on-top-of-a-modal; the component was deleted, its logic moved inline |
| **4.8.0** | **Visual/UX polish pass** (B40, B41, B42, B37 — owner feedback, 2026-07-09). **B40**: sidebar bottom rail redesigned as a 2×2 icon+label grid (Changelog/Deleted top, Settings/Sync bottom) — the 1×4 icon-only row from v4.6.5 traded away readability once tooltips turned out not to be enough. **B41**: Focus's picker now fills the full available width as a flex-wrapped "corkboard" of varying-size note cards (3 size tiers, deterministic per-card tilt hashed from task id) instead of a capped-width list; suggested tasks render largest with an accent border; the locked/committed list stays a plain list. **B42**: removed Agenda's `max-width: 900px` cap. **B37**: the Android widgets were fully rebuilt — first a visual pass on the existing 3 separate widgets, then (after live on-device feedback that 3 cards is inherently the problem, not the palette) collapsed into **one combined widget**, then simplified further to 3 static buttons with zero dynamic data per direct owner direction ("no text, just 3 buttons") — a wide accent-colored "Focus" pill plus 2 small circles (Quick Add, Dashboard). The old `OffologWidgetPlugin`/`widgetBridge.ts` data-push pipeline was deleted outright since a static widget has nothing to push. A real structural bug was found and fixed along the way: the widget kept rendering stretched regardless of `minWidth`/`resizeMode` tuning, because the card background was painted on the widget's full allocated launcher-grid frame (which can be much bigger than the declared minimum) rather than on an inner `wrap_content` child — fixed by making the outer frame transparent and only the inner content card visible, the same pattern most polished Android widgets use. Final on-device visual sizing is an open, owner-driven polish pass — not scheduled to a future version, picked up directly in Android Studio |
| **4.7.0** | **Perf validation + import/export v2 + Agenda week view** (A10, A24, B4, B7). **A24**: new `tests/perf.bench.ts` + `npm run bench` (Vitest's native `bench`, kept separate from `npm test`) timing `getDashboardData()`, `getTasksForProject()`, and `searchAllTasks()` — no hardcoded thresholds, compare printed numbers release to release. **A10**: ran it at a 3,000-task/8-project stress scale — `getDashboardData()`/`searchAllTasks()` stayed sub-millisecond (in-memory cache), `getTasksForProject()` (the one path hitting PouchDB's Mango index directly, uncached) sat around 10ms; no perf cliff found. **B4**: export a single project (JSON, via a project picker in Settings → Data), export all tasks as CSV (one-way, not re-importable), and a guided import — pick a file, see a preview of what will be created per doc type (and what's unrecognized) before confirming, instead of writing immediately on file-select. **B7**: Agenda gets a List/Week toggle (per-device) with a 7-column week grid, prev/next/Today navigation, and priority-colored task bars; the first version boxed each task in its own bordered chip and was cramped/hard to scan, redone as a plain compact per-day list. A real reactivity bug was caught and fixed along the way: the week grid never updated once its data finished loading async, because a plain function call inside the `{#each}` block didn't register `all` as a tracked dependency — fixed with an explicit `$:` lookup |
| **4.6.5** | **UI polish patch** (owner feedback, 2026-07-09) — no schema/logic changes, purely Sidebar/CardDetail/List UI. **Sidebar**: Dashboard/Focus/Agenda unified into one lighter nav style (was 3 separately-bordered pills); Spaces and Projects merged into a single collapsible tree instead of two separate flat lists (real bug fixed along the way: the currently-active space couldn't be collapsed — a reactive "keep active space expanded" statement was re-triggering itself on every collapse); sync status compacted from a bordered sentence row into a small icon in the bottom rail (Changelog/Deleted/Settings), with a status dot + conflict-count badge and full detail in a tooltip; that bottom rail also switched from icon+text-label buttons (unreadable once a 4th button was added) to icon-only with tooltips. **CardDetail**: Status/Priority merged into a 2-column row, the "remind on due date" toggle+hint nested under the Reminder field instead of floating as separate rows, tighter padding/type scale throughout, section dividers for grouping. **List view**: custom fields (B16) now appear as toggleable columns, not CardDetail-only (real bug fixed: the Columns popover was silently clipped by `.list-panel`'s `overflow:hidden` once enough rows existed to overflow it — re-anchored with `position:fixed` computed from the button's position). **New**: `CalendarPicker.svelte` — a fully custom, theme-matched calendar + time picker (B38) replacing the native `<input type=date>`/`<input type=datetime-local>` in CardDetail's Due date and Reminder fields |
| **4.6.0** | **Appearance + power-user pass** (A11, B21, B11, B16, B19) — merged from two originally-separate rows (owner request) into one release. **B21**: theme is now Light/Dark/System (was a boolean toggle) — System follows `prefers-color-scheme` live via a watcher, with one-time migration from the old `dark` localStorage key so an explicit prior choice isn't silently flipped. **B11**: a High Contrast toggle, layered on top of Light/Dark via `body.high-contrast`/`body.dark.high-contrast`, raising border/text contrast only (not the meaning colors). **B16**: custom fields (text/number/date/select) — shipped **global**, not per-project as originally scoped, after owner feedback mid-build that per-project fields with inline add/remove in `CardDetail` invited sprawl; now managed once from Settings → Organize → Manage Custom Fields, stored as a single `meta:custom_fields` doc, with `CardDetail` capping visible fields to 3 by default (a "show more" toggle) so cards don't balloon. **B19**: bulk actions in List — also revised twice from the original design: no shift-click range-select (dropped), row checkboxes only appear in an explicit "Select" mode merged into the existing Columns dropdown (a toggle switch, not a separate always-visible button), custom-styled checkboxes (not the native browser box), and the bulk bar trimmed to move-to-status / change-priority / add-tag only (no remove-tag, no bulk delete). **A11**: audited every mutation call site touched by this release (custom fields, bulk actions) for the try/catch+`showError()` invariant, plus a spot-check of existing surfaces — no gaps found |
| **4.5.0** | **Focus view (B35), draft.** New global nav item beside Dashboard/Agenda. Not the originally-scoped auto-computed priority list — that design was built, then rejected live by the owner ("not making sense, we need a really focus space") for feeling too passive/similar to Agenda. Replaced same-session with a **daily commitment lock**: pick up to 3 open tasks each day into a commitment; only those show until each is done or the day rolls over. The picker ranks candidates (pinned > overdue > due-soon > priority) but round-robins across those reason-buckets so the 3 suggestions are a genuine spread rather than "the 3 most overdue tasks" every morning; suggested rows are visually highlighted and labeled with a colored "why" chip (Pinned/Overdue/Due soon/High priority). The lock itself (`{date, taskIds}`) lives in `localStorage['offlog_focus_lock']`, deliberately not a PouchDB doc — ephemeral, per-device, unsynced. Commit button is a sticky footer, not buried at the bottom of a scrolling list. Still an open draft: add-task-from-Focus, a Dashboard link, and a Samsung-Daily-Brief-style summary card are confirmed next steps, not yet built (see ROADMAP.md's B35) |
| **4.4.2** | **Maintenance pass** (second run, after v4.4.0/v4.4.1 shipped, per the owner-set schedule in MAINTENANCE.md). Phase 0/1 baseline and analysis found the codebase largely clean — same conclusion as the first pass; approved fixes: `QuickAdd.svelte`'s `doAdd()` had no try/catch/`showError()` around `createTask()` (the one real gap in the otherwise-audited invariant — a failed write left "saving" stuck true forever with no error shown), and `App.svelte`'s `setView()` write (now-vestigial `default_view` persistence, per A27) got a documented silent catch, matching the existing `rescheduleAll().catch(() => {})` "fire and forget" pattern rather than a misleading error toast for a background write that doesn't affect the already-succeeded view switch. `npm audit`: unchanged (20 vulnerabilities, all dev/build tooling, none reach the shipped app). No dead code, no unused deps, cache-invalidation coverage confirmed complete |

Older releases (v4.4.0 and earlier): [CHANGELOG-ARCHIVE.md](CHANGELOG-ARCHIVE.md).
