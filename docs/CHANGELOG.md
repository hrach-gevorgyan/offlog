# Offlog — Changelog

The single source of truth for version history — this used to be duplicated
across `TECH.md` (technical detail) and the root `README.md` (short
highlights), which had drifted in coverage (TECH.md's table was more
complete). Consolidated here 2026-07 as part of the documentation cleanup;
both `README.md` and `docs/TECH.md` now link here instead of keeping their
own copy. New entries go at the top, one row per released version.

This table keeps the newest ~8 releases in full detail. Older releases are
compressed to one line each in [CHANGELOG-ARCHIVE.md](CHANGELOG-ARCHIVE.md)
(git still has every full diff) — split out 2026-07 once this file's
default-loaded size grew large enough to matter. When adding a new entry,
move the oldest row here into the archive (compressed to one line) so this
table stays roughly the same length release over release.

| **4.22.2** | **Sync stability fixes** (E2 — owner-reported "not stable yet," 2026-07-20). Found debugging a corrupted CouchDB shard on the dev machine, all three from the same root-cause family — the app trusting a stale identity instead of re-checking it. **E2**: the phone stored a fixed IP:port at pairing time and never revisited it, so any DHCP lease renewal or router reboot silently broke sync until a manual re-pair. Now persists the CouchDB server's own stable `uuid` (already broadcast over mDNS, already returned by pairing) and re-resolves it automatically on a "Cannot reach sync server" failure, updating the stored URL and retrying — no more manual re-pairing for an IP that just moved. **Dev/prod Tauri identity collision**: `cargo tauri dev` and a real installed build shared the exact same `sync-host.json` (port/credentials) despite normally running against different CouchDB data — a phone could end up silently talking to the wrong build's database. Debug builds now get their own config file and, if the usual per-target dev CouchDB copy is ever missing, their own isolated bootstrap copy instead of ever running live out of the one shared `vendor/couchdb-win` source directory. **Android debug/release storage collision**: a debug APK shared the same `applicationId` as a real install, overwriting/sharing its storage on every Android Studio test run — debug builds now install side by side via `applicationIdSuffix ".debug"` |
| **4.22.1** | **Maintenance pass** (eighth run, after v4.22.0 shipped, per the owner-set schedule in MAINTENANCE.md — first pass since v4.19.1, covering v4.20.0/v4.21.0/v4.22.0's changes). Found and fixed one real gap: `ChangelogView.svelte`'s "Clear all" button called the mutating `clearLogs()` with no try/catch/`showError()`, unlike every other mutation site (an audited invariant since v2.9.0) — fixed. Also added a Tauri window `minWidth`/`minHeight` (640×480) to `offlog-desktop`'s `tauri.conf.json`, since the window previously had no floor on how small it could be resized. Everything else checked out clean: no dead code, no new duplication, `npm audit` unchanged (same 4 dev-tooling-only advisories, none reachable from the shipped bundle), every `{@html}` site still correctly escaped/fixed-constant, no `eval`/`new Function`/stray `innerHTML=`, `offlog-desktop`'s `unsafe` blocks limited to the expected `TerminateJobObject` FFI calls, `pairing.rs` never logs credentials by value. One correction from the initial audit: `pouchdb` (the npm package, not `pouchdb-find`) looked unused in `src/` but is actually imported directly by `tests/setup.ts` as the real PouchDB stand-in during tests — left in place, removing it would have broken `npm test`. Disabled CSP (`security.csp: null` in `tauri.conf.json`) noted but deliberately not touched this pass — bundling it into the same pre-public-release security pass as C7 instead of a piecemeal maintenance fix, since enabling it needs real testing against the app's inline styles and dynamic-port sync fetches |
| **4.22.0** | **Week-start-day setting + full animation pass** (B47, B51 — owner-directed, 2026-07-20). **B47**: Settings → Appearance gets a Sunday/Monday "Week starts on" toggle, wired into Agenda's week grid and `DeadlinesView.svelte`'s "this week" grouping (both previously assumed a fixed Sunday start). Timezone, the other half of B47's original scope, deliberately not built — the app's local-time-only design is correct for a single-device personal task manager, see ROADMAP.md's B47 entry. **B51**: owner feedback that many actions "snapped" with no animation, reading as fake — an inventory pass found the real pattern was that things which *open* mostly animated already, but almost nothing that *closed* did, plus several everyday actions (checklist toggle, card create/delete/archive, the new v4.21.0 "⋯" menus) had no transition in either direction. New `src/lib/motion.ts` centralizes shared transition parameters (some panels/toasts needed *custom* transition functions, not Svelte's built-ins — those clobber an element's own `transform: translate(-50%,-50%)` centering mid-animation); every panel, dialog, popover, toast, card, and column that was missing open/close animation now has one. Also fixed, found in the same pass: `offlog-desktop`'s Tauri WebView was intercepting native OS drag-drop by default, silently breaking the Kanban board's HTML5 drag-and-drop on the PC app — `dragDropEnabled: false` in `tauri.conf.json` fixes it (no separate `offlog-desktop` build/sync step needed for the web-side changes here — `frontendDist` points straight at `offlog-app/dist`, so a normal `npm run build` updates both apps at once) |
| **4.21.0** | **Card Detail redesign + Kanban quick-actions menu** (B49, B53 — owner-directed, 2026-07-19, mockup-validated before implementation per the List/Table merge precedent in DECISIONS.md). **B49**: Due date and Reminder — previously two always-visible fields with their own shortcuts/checkbox/hint sprawl — now collapse into one labeled, clickable "Schedule" row showing a live summary (e.g. "Wed, Jul 15 · 9:00 AM reminder") that expands to the same two fields, unchanged. Checklist/Custom fields/Notes restyled as consistent bordered card-rows (icon + label + chevron) instead of three independently-styled accordions. Delete/Archive/Duplicate plus the history toggle — 4 separate footer controls — consolidated into one "⋯" dropdown (History, Archive, Duplicate, then Delete set apart in red), reusing `CustomSelect.svelte`'s click-outside-to-close pattern. Created/Updated timestamps were tried inside the menu too but dropped after owner feedback — once History is open the panel itself already shows that info, making the static text redundant. Every field and function verified identical to before — this was a pure layout reshape. **B53** (new, folded in mid-review at the owner's request): Kanban cards gain their own per-card "⋯" quick-actions menu — Pin/Unpin, Archive, Duplicate, Delete directly from the board, no need to open Card Detail. Hover-revealed on desktop, always-visible at ≤768px (same touch-fallback pattern as the existing column-action buttons); writes immediately rather than batching into a form save, since a Kanban-level action should take effect the moment it's clicked |
| **4.20.0** | **Export/import redesign + first-run device-name prompt** (B45, B46 — owner, 2026-07-13). **B45**: Settings → Data's four loosely-related buttons (Export JSON, Export CSV, Export Project, Import JSON) redesigned into two clear groups — **Back up** (one scope control, "Everything" or pick a project, feeding a single "Back up" button; CSV kept separate, clearly labeled "one-way, can't be restored") and **Restore** (Import JSON, unchanged). `doBackup()` merges the old `exportJSON()`/`doExportProject()` into one function branching on scope. **B46**: added `NamePrompt.svelte`, a lightweight modal (reuses `ConfirmDialog.svelte`'s pattern) asking for this device's name once, ever, right after the app becomes usable — never gates getting in. Skip and Save are equal-weight actions; the "shown once" flag is set immediately regardless of which is chosen, since skipping is as valid a choice as naming it (C2's zero-config-first-run principle) |
| **4.19.1** | **Maintenance pass** (seventh run, after v4.19.0 shipped, per the owner-set schedule in MAINTENANCE.md — first pass to cover `offlog-desktop/` alongside `offlog-app/`). Found and fixed one real duplication: `isTauri()` detection and the raw `window.__TAURI_INTERNALS__.invoke(...)` call pattern were each independently re-declared/inlined across `config.ts` and `SettingsPanel.svelte` (5+ call sites, each with its own `as any` cast) — consolidated into `isTauri()`/`invokeTauri()`, both now exported from `config.ts`. Everything else checked out clean: no dead code beyond one already-documented pending stub (`SyncHostInfo::url()` in `sync_host.rs`), no unused dependencies added by Track E (`mdns-sd`, `tiny_http`, `win32job`, `capacitor-zeroconf`, `@capacitor/filesystem`, `@capacitor/share` all genuinely used), `npm audit` unchanged from prior passes (`uuid` via `pouchdb-find` remains the one genuinely-reachable advisory), all `unsafe` Rust blocks are the expected `TerminateJobObject` FFI calls only, no credential values found in any log line, and the deep-link handler / `{@html}` XSS surface are both still correctly hardened from earlier passes. `SettingsPanel.svelte` flagged again as oversized (1141 lines, grew from Track E's pairing UI) but a split was deliberately skipped — same shared-CSS-duplication blocker that stopped the v4.12.0 pass's attempt still applies |
| **4.19.0** | **Track E working end-to-end + C7/C2/C10 progress** (2026-07-14 to 2026-07-19). Track E (the PC standalone app, `offlog-desktop/`, Tauri) went from a scoping plan to fully working: an embedded CouchDB sync host that self-configures with a random per-install port/credentials, mDNS discovery, a single-use expiring-code pairing handshake to get real credentials onto a phone safely, a real NSIS installer, and Windows Job Object hardening so the whole process tree dies with the app on any exit path. About a dozen real bugs found and fixed along the way — the biggest was that the Tauri app itself was never pointed at its own embedded sidecar, silently talking to an unrelated CouchDB instead (`initTauriSyncDefaults()` in `config.ts` is the fix); full detail lives at each fix site as code comments, and a consolidated summary is in `docs/ROADMAP.md`'s Track E entry rather than repeated here. **C7**: removed the hardcoded CouchDB password fallback from `config.ts` entirely — falls back to `''` (not-configured) now, same as `DEFAULT_SYNC_URL` already does; git history still needs handling at C1 (going public) time, tracked explicitly so it isn't mistaken for fully closed. **C2**: Android's first-run "not connected" message now points at the actual easy path ("Find my computer") instead of the technical "Developer options" label. **C10** (plain-language sweep, partial): fixed a real naming inconsistency (sidebar called it "Deleted," the panel itself is titled "Recycle" — unified to "Recycle"), a broken/ungrammatical aria-label ("Select and columns" → "Select rows or show/hide columns"), and replaced "database" jargon in the Maintenance tab's step labels and hint text with plain descriptions of what each step actually does. Installer signing deliberately deferred (owner decision — real annual cost, not required for Track E to be functionally done, revisit only if it becomes real user friction) |
| **4.18.0** | **Sync/Android reliability release** (A32, A33, A34, A35 — owner on-device testing, 2026-07-13; bumped ahead of the plain-language pass). **A32**: sync silently reported "synced" when it wasn't. Root cause: PouchDB's combined `db.sync()` wrapper discards the error its `push`/`pull` sub-replications carry when either pauses to retry — `attachSyncHandlers()` (`db.ts`) now listens to `handler.push`/`handler.pull` directly to recover it. Also fixed the actual infrastructure gap this had been masking: CouchDB's `bind_address` defaulting to localhost-only, a stale LAN IP, and Android's `network_security_config.xml` hardcoding cleartext permission to one specific IP instead of a broad allowance. **A35**: `config.ts`'s `DEFAULT_SYNC_URL` no longer falls back to a hardcoded real address (the actual cause of the "worked, then broke after reinstall" regression) — on desktop web it now defaults to `http://127.0.0.1:5984/offlog` (loopback is structurally correct for this app's "PC is the host" architecture and can't go stale the way a remembered LAN IP can); scope was narrowed from an original 3-phase auto-discovery plan after the owner concluded the phone-side pieces would be thrown away once Track E (native PC app) ships — see ROADMAP.md. **A33**: Android reminders fired silently — nothing ever created a notification channel, so Android fell back to an auto-created one with unreliable importance; fixed with an explicit high-importance `reminders` channel, created up front and tagged on every scheduled notification. **A34**: Export JSON/CSV/Project did nothing on Android — the blob-URL `<a download>` trick has no download manager inside a Capacitor WebView; fixed by writing to app storage and handing off to the OS share sheet on native, via two new dependencies (`@capacitor/filesystem`, `@capacitor/share`, owner-approved) |
