# Offlog — Changelog

The single source of truth for version history — this used to be duplicated
across `TECH.md` (technical detail) and the root `README.md` (short
highlights), which had drifted in coverage (TECH.md's table was more
complete). Consolidated here 2026-07 as part of the documentation cleanup;
both `README.md` and `docs/TECH.md` now link here instead of keeping their
own copy. New entries go at the top, one row per released version.

This table keeps the newest ~8 releases in full detail. Older releases are
compressed to one line each in [CHANGELOG-ARCHIVE.md](CHANGELOG-ARCHIVE.md)
(git still has every full diff) — split out 2026-07 once this file's
default-loaded size grew large enough to matter. When adding a new entry,
move the oldest row here into the archive (compressed to one line) so this
table stays roughly the same length release over release.

| Version | Changes |
|---|---|
| **4.15.0** | **Housekeeping release** (A9, B24, B29). **A9**: real `.svelte` component test coverage begins — `@testing-library/svelte` added (new dev dependency), first slice covers `CardDetail`'s save logic (5 tests: edited-field persistence, reload-after-save, error-toast-on-failure, due-date quick-shortcut, checklist-toggle-persistence). Needed one Vitest config fix along the way: without `resolve: { conditions: ['browser'] }`, Vite/Vitest resolved `svelte` via its default (SSR/Node) export condition even under the jsdom test environment, hard-failing any component using `onMount`/`onDestroy` — this is the first real component test, so nothing had exercised that path before. `KanbanBoard`'s drag/drop math and `Sidebar`'s Maintenance orchestration (the roadmap's other two named targets) remain uncovered — real HTML5 drag events and the Maintenance flow's much larger dependency surface are both bigger lifts, left for a follow-up pass. **B24**: default seed (`seedIfEmpty()`) drops from 4 spaces to 3 — Unsorted/Personal/Work, no longer Family (still creatable manually via "+ New space"). **B29**: tags now render on Kanban cards (compact chips, same visual treatment as List's existing `.tag` chip, duplicated as a separate rule since Svelte scopes styles per-component) — previously only visible in List view |
| **4.14.0** | **Project templates + notes guardrail** (B8, B30). **B8**: "+ New project" gains a "Use a template…" step — pick an existing project to copy its status/column structure into the new one (fresh column ids, same names), with an optional "Also copy open tasks" checkbox that copies non-completed tasks (positional-last-column check, same as everywhere else), resetting due dates/reminders/checklist-done-state on the copies since a template's deadlines and progress don't apply to a fresh instance. New `createProjectFromTemplate()` in `db.ts`, tested (`tests/db.test.ts`) alongside the existing CRUD suite. A real bug was caught mid-build: the template picker's own buttons blurred the project-name input before their `on:click` fired (blur precedes click in the browser's event order), silently submitting a blank project — fixed with `on:mousedown|preventDefault` on those buttons so they don't steal focus in the first place. **B30**: `CardDetail`'s notes field shows a small character counter once past 500 characters — a soft nudge, not a hard block; notes stay unbounded markdown |
| **4.12.2** | **A31 visual/UX pass, part 1** (mobile + dark-mode spot-check, same-day follow-up to v4.12.1). Two real bugs found and fixed: **Dashboard's project grid overflowed at mobile widths** (375px), clipping card titles past the viewport edge with no way to scroll to them — root cause was `.dash-cols`'s mobile media query using a bare `grid-template-columns: 1fr`, whose implicit minimum is `auto` (= its content's max-content size) rather than actually clamping to the container, so the single column grew to 405px inside a 337px parent; fixed with `minmax(0, 1fr)`. Chased this through several wrong layers first (`.proj-card`, `.col-projects`, `.project-grid` `min-width`/`width` tweaks) before finding the actual root cause in `.dash-cols` itself — those intermediate attempts were reverted rather than left as redundant belt-and-suspenders CSS. **Focus view's fixed FAB overlapped its own Commit button** at mobile widths — both compete for the same bottom-right corner, since Focus is the one view with an additional in-flow action bar; fixed with right padding on `.fc-footer` sized to the FAB's footprint. Confirmed desktop and Kanban/List/CardDetail/Settings unaffected at both mobile and dark-mode. A31 remains open — see its ROADMAP.md entry for what's still unvisited (several Settings tabs/managers, a full dark-mode pass, Android) |
| **4.12.1** | **UI audit fixes** (owner-directed accessibility + design-system pass, 2026-07-12). **Accessibility**: `--faint` and `--accent` both failed WCAG AA contrast in one or both themes (`--faint` 2.37:1 light/3.60:1 dark; white-on-`--accent` button text 4.47:1 light) — retuned both tokens to clear 4.5:1+ everywhere, propagated to `capacitor.config.ts` and Android `colors.xml` per the brand-color rule. **Design-system audit** surfaced a bigger, previously-missed instance of the same class of bug: ~17 components hardcoded `color: #fff` next to `background: var(--accent)` instead of using a token, which broke badly in dark mode (`--accent` is a *light* indigo there — white text measured 2.98:1, confirmed live via `preview_inspect` on the Settings theme toggle). Fixed with a new `--on-accent` token (white in light, `#181A20` in dark) applied at all 17 sites. Also consolidated `ChangelogView.svelte`/`TaskHistoryPanel.svelte`'s two independently-drifted `ACTION_COLOR` hex maps into one shared, theme-reactive export in `utils.ts` (`var(--success/--accent/--due-soon-ink/--danger)` instead of frozen hex) — required switching their translucent-pill-background trick from string-concatenating a hex-alpha suffix (breaks on a `var()` string) to `color-mix()`, per this repo's own token-tint convention. **UX**: added a CSS-only scroll-shadow affordance (`background-attachment: local/scroll` gradient trick, no JS) to Kanban's board, List's table, and Agenda's week grid, all of which overflow horizontally with no prior hint that more content exists off-screen; gave Card Detail's Delete button a distinct resting-state red tint instead of only-on-hover, so it doesn't read as equal-weight to Archive/Duplicate; scoped Settings' save button label to only say "Save & restart sync" on the Sync tab, plain "Save" elsewhere |
| **4.12.0** | **Full codebase audit + documentation-flow optimization** (A30 — owner-directed pivot away from feature work, 2026-07-12). **Code**: no dead code found. Fixed real duplication — `fmtLastSynced`/`fmtFullTimestamp` were byte-identical across Sidebar/SettingsPanel and CardDetail/TaskHistoryPanel, consolidated into `utils.ts`; `TrashView.svelte` reimplemented `timeAgo()` instead of importing the existing export. Fixed a real bug: `CardDetail`'s collapsible-section state (added in v4.11.1) was plain `let`, never reset per task — every one of its 6 call sites now wraps `<CardDetail>` in `{#key task._id}` to force a remount instead of risking stale state. Fixed a real accessibility gap: `CustomSelect.svelte` didn't move focus into its popover on mouse-click open, so arrow keys did nothing until manually tabbed in. `ListView.svelte`/`SettingsPanel.svelte` remain oversized (875/896 lines) — attempted to split both, but discovered their categories/sections all lean on ~150 lines of CSS classes defined once in the parent; extracting components now would mean duplicating that CSS into every new file, directly against this release's own goal — deferred pending a shared-style-layer decision. `npm audit`'s `uuid` advisory confirmed genuinely reachable (`pouchdb-find` is actually bundled, unlike core `pouchdb` which is UMD-only) but still not worth a breaking downgrade. **Docs**: ROADMAP.md's own "shipped items get a one-line pointer, full detail in CHANGELOG.md" rule was being ignored by ~15 items carrying full duplicate paragraphs — trimmed, cutting the file from 607 to under 450 lines. CLAUDE.md's doc-list synced to mention MAINTENANCE.md/CHANGELOG-ARCHIVE.md; checked for PWA-removal staleness (clean, no gaps). Full findings in ROADMAP.md's A30 entry |
| **4.11.1** | **Design-kit polish + PWA removal** (owner feedback, 2026-07-12). **Focus view**: went through two rounds of visual iteration — first a priority-tinted "sticky note" look with a washi-tape accent (too playful), then a priority-colored left border stripe (redundant with the existing priority dot — two signals for one fact), settled on plain solid cards with the tilt/size corkboard layout unchanged and color left to the priority dot alone. **Design kit**: every remaining native `<select>` across the app (CardDetail's Status/Priority/custom-field pickers, FilterBar's Status/Tag filters, List's bulk-action selects, Settings' Export/Archive/Custom-Field-type pickers) converted to the existing themed `CustomSelect.svelte`, which gained a `disabled` prop for this; a real robustness bug was caught and fixed in `FilterBar.svelte`'s outside-click handler along the way (assumed `e.target` was always an `Element` with `.closest()`, which isn't guaranteed). **CardDetail**: Checklist/Custom Fields/Notes are now collapsible sections (collapsed by default only when empty — a task with existing content in one starts expanded) instead of always-open, addressing the page feeling overloaded once Checklist landed in v4.11.0; Status/Priority/Due/Reminder/Tags stay always-visible as the task's core identity. **PWA removed entirely** (`vite-plugin-pwa` uninstalled, manifest/service-worker generation dropped from `vite.config.ts`) — resolves the long-standing A26 staleness problem by removing the subsystem rather than patching it further; `main.ts` now actively unregisters any leftover service worker from a previous PWA-enabled build so an already-installed browser doesn't stay stuck on a stale cached version. Web is a plain always-fresh browser tab again; Android (Capacitor) is unaffected. See DECISIONS.md for the full rationale and QUESTIONS.md's new Q6 on the still-open "real PC standalone app" direction |
| **4.11.0** | **Kanban filters + checklists** (B2, B18). **B2**: List's filter popover + saved-filters logic extracted into shared `FilterBar.svelte` so Kanban gets the identical feature (saved filters are per-project, not per-view — one saved from List shows up in Kanban too). Placement went through two rounds of owner feedback: a dedicated Kanban toolbar row wasted space for a single button, so the Filters button now lives in the shared board-header, paired with the existing Search (Ctrl+K) button into one compact icon-only pill via a new `compact` prop, instead of either a lone button or two loose ones. A real bug was caught extracting the popover into its own component: it used to lean on `position:absolute` anchored to a wide ancestor (`.toolbar`) both for its width calc and to dodge that ancestor's `overflow:hidden` clipping — moving it into `FilterBar.svelte` broke both assumptions (clipped when the task list was short, pushed off-screen on mobile). Fixed with `position:fixed` computed from the button's own `getBoundingClientRect()` on open, the same pattern already used for List's Columns popover (v4.6.5). **B18**: `TaskDoc.checklist?: { text; done }[]` — flat, not nested/reorderable. Edited in `CardDetail` with the same add/remove/toggle interaction as tags, batched into the existing `save()` rather than a per-toggle write. A "☑ N/M" progress badge (turns `--success`-colored once complete) surfaces on both the Kanban card and the List row's title cell |
| **4.10.1** | **Maintenance pass** (fourth run, after v4.10.0 shipped, per the owner-set schedule in MAINTENANCE.md). Phase 1 found a real stored-XSS gap: `GlobalSearch.svelte`'s `highlight()` wrapped search-query matches in `<mark>` but never HTML-escaped the underlying text before rendering it via `{@html}` — since task titles can arrive via CouchDB sync from another device, a synced title like `<img src=x onerror=...>` would execute unescaped in Ctrl+K's results; fixed by escaping first, then matching/wrapping against the escaped string. Also found `archiveProject()`/`unarchiveProject()` (new in v4.9.0) never called `logChange()`, unlike every other project mutation — added, mirroring `updateProject()`'s existing `'update'`-action log shape. Two small hygiene fixes: a stray "column" in a Kanban tooltip that should've said "status" per the app's own wording convention, and a stale comment referencing the deleted `MaintenanceModal.svelte` by name. `npm audit`: 20 advisories, 19 dev/build-tool-only; the one reachable-from-runtime advisory (`uuid`, via the pouchdb dependency chain) needs a breaking pouchdb downgrade to fix — flagged, not applied. No dead code beyond the one stale comment, no unused deps; `ListView.svelte`/`SettingsPanel.svelte` file-size and duplicated timestamp-formatting logic (6 files) noted as split/extract candidates for a future pass, not touched this cycle |
