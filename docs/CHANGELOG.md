# Offlog — Changelog

The single source of truth for version history — this used to be duplicated
across `TECH.md` (technical detail) and the root `README.md` (short
highlights), which had drifted in coverage (TECH.md's table was more
complete). Consolidated here 2026-07 as part of the documentation cleanup;
both `README.md` and `docs/TECH.md` now link here instead of keeping their
own copy. New entries go at the top, one row per released version.

This table keeps the newest **10** releases in full detail. Older releases
are compressed to one line each in
[archive/changelog-archive.md](archive/changelog-archive.md) (git still has
every full diff; that folder also holds the full maintenance-pass log,
moved out of MAINTENANCE.md 2026-07-20) — split out 2026-07 once this
file's default-loaded size grew large enough to matter. When adding a new
entry, move the oldest row here into the archive (compressed to one line)
once this table exceeds 10 rows, so it stays roughly the same length
release over release. Archive weekly, or whenever it's due, not on a fixed
per-release schedule.

| **5.7.9** | **Real bugs from live-testing v5.7.6/5.7.7's new updater/quiet-hours features**, plus a batch of smaller fixes. Quiet hours: queued reminders all computed to fire at the exact same window-end instant — a real backlog could hit Android/Windows notification queues as a simultaneous burst, some of which throttle/collapse bursts rather than showing each one; `notifications.ts`'s `applyQuietHours()` now takes a `staggerIndex` (15s apart per reminder) applied across every scheduling path, 2 new tests. Kanban drag-and-drop: touch-drag was writing into `dragTask`, the same variable the desktop mouse-drag path guards on — if a touch sequence ever ended without `touchend`/`touchcancel` firing (OS taking over the gesture for its own edge-swipe/scroll handling mid-drag), `dragTask` stayed stuck non-null with nothing left watching it, fixable only by a full app restart. Touch and mouse drag now track fully separate state, plus a 15s watchdog and a document-level capture-phase failsafe as defense in depth. Desktop console flash on launch: the CouchDB sidecar spawn already had `CREATE_NO_WINDOW` set, but `couchdb.cmd` is CouchDB's own upstream launcher script and can get a nested child to request its own console regardless of the parent's flags — redirecting all three standard handles to null (the standard mitigation) added in `sync_host.rs`. Also this round: `CustomFieldManager`'s field-type dropdown opened downward and off-screen (always the last row in that modal) — now opens upward like `QuickAdd`'s project picker already does; Settings' Software Updates section tightened (version + check-for-updates merged into one row); `tauri.conf.json` gained `publisher`/`copyright`/`installMode: currentUser` so the installer shows real branding instead of placeholder text and skips the UAC elevation prompt |
| **5.7.8** | **Test release only** — a version bump with no functional change, cut purely to exercise the new updater UX (v5.7.7) a second time end-to-end from a real installed app (check → changelog modal → download progress → explicit restart prompt) |
| **5.7.7** | **Desktop updater UX overhaul** (E3 follow-up, prompted by the first real end-to-end update test in 5.7.6 surfacing how invisible the whole flow was). Settings now shows the running app version (desktop via `@tauri-apps/api/app`'s `getVersion()`, mobile via `@capacitor/app`'s `App.getInfo()` — mobile gets version display only, no update button, since updates there go through the Play Store once C3 ships). New `updateChecker.ts` splits the old single `downloadAndInstall()` call into `check()`/`download(onEvent)`/`install()` (confirmed available on `@tauri-apps/plugin-updater`'s own `Update` type) so the UI can show a real download percentage instead of a blind wait, and so installing is never a surprise: after downloading, a new `UpdateModal.svelte` shows the release notes (see below) and an explicit "Restart to install" / "Later" choice — auto-restart is gone. A silent background check runs every ~6h on desktop and surfaces a small dismissible banner when an update is found, with its own Settings toggle to turn it off entirely for anyone who wants to stay on a version they're happy with (manual "Check for updates" always still works regardless). `latest.json`'s `notes` field was always empty before — `release.yml` now extracts the current version's own CHANGELOG row (new `.github/scripts/extract-changelog-entry.js`) and passes it as `tauri-action`'s `releaseBody`, so the modal's changelog text is the real thing, not a placeholder |
| **5.7.6** | **Quiet hours + a real 12h/24h display bug fix.** New Settings → Notifications → Quiet Hours: reminders due inside a configured local window (default 22:00-07:00) queue until it ends instead of firing — `notifications.ts`'s new `applyQuietHours()` applies to every scheduling path (web `setTimeout`, Tauri desktop toast timer, Android's `LocalNotifications.schedule`) and to catch-up firing (a reminder that comes due while quiet hours are already active on app reopen also queues, rather than firing immediately); 6 new tests in `notifications.test.ts` (suite now 210). Also fixed, same root cause across three spots: `TimePicker.svelte`'s hour dropdown always rendered 24h (`00`-`23`) regardless of the Settings → Appearance 12h/24h choice — now shows a real 12h + AM/PM picker in that mode; `CalendarPicker.svelte`'s date+time trigger label and `CardDetail.svelte`'s "Remind me on the due date at HH:MM" text had the same bug, both now route through `fmtTime()` like every other clock display in the app. Found via the first-run onboarding flow (`NamePrompt.svelte`'s "quick preferences" step) specifically, which is also where its own toggle's example-time labels (`1:00 PM`/`13:00`) were replaced with plain `12h`/`24h` to match Settings' own wording, since the toggle's buttons aren't reminder timestamps and don't need to demonstrate the format. This is the first release exercising the real signed update path end-to-end (E3, shipped in 5.7.5) |
| **5.7.5** | **Milestone 1 complete: B61, B62, E3.** B61 — changing/removing the App Lock PIN now requires the current PIN first (new `ConfirmPinGate.svelte`); the old confirm-dialog on removal was dropped since entering the PIN is itself the confirmation. B62 — automatic local backup: on app start, at most once a day, a full JSON snapshot is written to app-private storage and rotated to the newest 7 (desktop: `appDataDir()/auto-backups/`; Android: `Directory.Data/auto-backups/`; no-op on web) — Settings gets a short, plain-language status line, full technical detail moved to TECH.md rather than cluttering the menu. E3 — the desktop auto-updater is wired for real: `tauri.conf.json`'s placeholder pubkey/endpoint replaced with a real signing key and the GitHub Releases `latest.json` URL; `release.yml` now signs the Windows installer and produces `latest.json` whenever the two Tauri signing secrets are present, falling back to an unsigned build otherwise (same safety pattern as Android's keystore). Also: A32's UI test suite (18 new component tests) landed the same batch, bringing the total to 204. This is the first release built to actually test the new signed-update path end-to-end |
| **5.7.4** | **Second dependency maintenance batch**, same day as 5.7.3 — 6 more Dependabot PRs (v5.7.3's tag had already been cut when these landed, so folded into a new bump rather than reopening that release). `@capacitor/cli` 8.4.1→8.4.2, `@capacitor/core` 8.4.1→8.4.2, `@capacitor/app` 8.1.0→8.1.1, `@capacitor/local-notifications` 8.2.0→8.2.1, `serde_json` 1.0.150→1.0.151 — all low-risk patch releases, verified with a real `cargo build` where applicable. `rand` 0.9.5→0.10.2 (offlog-desktop) needed a real fix, not just a version bump: `random_range` moved from the `Rng` trait to a new `RngExt` trait in 0.10, breaking `sync_host.rs`'s pairing-credential generator (the port/user/password/cookie the embedded CouchDB host uses) — fixed by importing `RngExt` instead, pushed directly onto the PR's own branch and verified with `cargo build` before merging. | **5.7.3** | **Dependency maintenance batch** — first real exercise of the CI/Dependabot pipeline set up in 5.7.2. Notably: `uuid` forced to 11.1.1 via a new npm `overrides` entry (GHSA-w5hq-g745-h8pq, a moderate buffer-bounds-check issue in `v3()`/`v5()`/`v6()`; transitive via `pouchdb`/`pouchdb-find`/`pouchdb-adapter-memory`, none of which had bumped it themselves yet — `overrides` forces the patched version across the whole tree without waiting on upstream). TypeScript's 6.0.3→7.0.2 bump was merged after a clean local build/`tsc --noEmit`/test pass, then reverted the same day: it broke `npx cap sync android` in the release pipeline's `build-android` job (Capacitor CLI's own config loader, not app code) — none of the local checks actually exercise `cap sync`, so this failure mode was invisible until the real release build. | **5.7.1** | **CRITICAL — real dev credentials were baked into production builds** (owner live-testing on Android, 2026-07-21: onboarding never offered sync setup on a fresh install, and the sidebar showed a 4th sync button with an error state despite never pairing). Root cause: Vite loads `.env.local` for every build mode, not just `npm run dev` — the developer's own `.env.local` (real self-hosted CouchDB URL + username + password, meant purely for local dev convenience) was compiled directly into `dist/`, and from there into the just-published v5.7.0 GitHub Release's Android APK, in plaintext. Confirmed: the real password was recoverable straight out of the shipped APK. Fixed by gating `config.ts`'s `envUrl`/`envUser`/`envPass` reads on `import.meta.env.DEV`, which Vite resolves statically at build time — a production build now has these as literal `undefined`, so the minifier removes the actual secret values from the bundle entirely rather than just skipping their use at runtime. Verified: rebuilt `dist/` no longer contains the credential anywhere. The v5.7.0 Release (and its leaked APK) was deleted; the real CouchDB password needs rotating separately since it left git's reach entirely (distributed in a binary file, not through the repository — `dist/` itself was never committed). Process gap closed: `docs/MAINTENANCE.md`'s security checklist now explicitly requires checking the actual built `dist/` output for `.env.local` leakage every pass, not just source files and git history — a source-only secret scan structurally cannot catch this class of bug. **Also from the same live-testing session** (this version wasn't released yet, so folded in rather than a separate bump): widget taps not switching views on a warm start on Samsung/OneUI (`FLAG_ACTIVITY_SINGLE_TOP` missing from the widget's `PendingIntent`); today/due-date math used UTC instead of local calendar date in 7 places (Agenda's "This week" view, `utils.ts`'s shared `TODAY()`/`dueLabel`/`dueRelative`, Focus's daily commitment lock, Global Search, Kanban's overdue badge, 2 export filenames) — consolidated to one shared `localDateStr()` (moved from `db.ts` to `utils.ts` to avoid a circular import); Kanban board didn't auto-scroll toward an off-screen column while dragging a card near the screen edge. **A third round, same still-unreleased version**: `isSyncEnabled()` defaulted true unconditionally, so a native device that skipped the onboarding sync offer still showed a misleading "Sync enabled" toggle in Settings — now defaults true only when a real `syncUrl` actually exists. Sidebar's bottom-row footer was a fixed 2-column grid (built for exactly 4 buttons) that left the 3rd button wrapping alone when sync's button is hidden — new 3-equal-column layout for that case. Sync tab and Advanced tab's manual-connection section now collapse to just the enable toggle when sync is off (same pattern App Lock uses), and the footer button only says "Save & restart sync" when sync is actually on. Time-format toggle relabeled from abstract "24h"/"12h" to concrete example times ("1:00 PM"/"13:00"); Privacy Screen's description rewritten to lead with what it actually does. Also this round: removed the unused `@capacitor/assets` devDependency (cleared 15 of 17 Dependabot advisories — it was pulling in the entire node-tar/xcode/minimatch chain), an unused `--dur-fast` CSS token, 2 orphaned icon assets, and a stale code comment |

Also this round: ROADMAP.md's header restated a stale version number instead of pointing at this file, the single source of truth — fixed and generalized as a rule.
